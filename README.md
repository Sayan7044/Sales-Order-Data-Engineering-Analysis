# Sales-Order-Data-Engineering-Analysis
Project Overview
This project simulates a real-world industrial data pipeline for the FMCG/Retail sector. It transforms messy, raw sales and customer records into an executive-level intelligence system. The project demonstrates the full lifecycle of a data analyst: Extract (Pandas), Transform & Feature Engineer (SQL), and Visualize (Power BI).

🏛️ Project Architecture
ETL Phase (Python/Pandas): Handled missing values, standardized categorical data, and addressed outliers in a messy dataset.
Database Phase (PostgreSQL): Loaded structured data into a relational database and performed advanced feature engineering using SQL Views.
Intelligence Phase (Power BI): Developed a Star Schema data model and DAX measures to track profitability and customer risk.

🛠️ Tech Stack
Language: Python 3.x (Pandas, NumPy)
Database: PostgreSQL (Advanced Joins, CTEs, Window Functions)
Visualization: Power BI Desktop (DAX, Star Schema Modeling)
Environment: Jupyter Notebook, pgAdmin 4

📈 Key Business Insights & KPIs
Revenue Optimization: Tracked Total Sales Volume and identified high-performing product categories.
Customer Segmentation: Engineered a Risk Analysis model using SQL to categorize customers (High, Medium, Low) based on credit and purchase history.
Profitability Analysis: Calculated Expected Interest Income and Profit Margins per region.
Operational Efficiency: Identified bottlenecks in the sales pipeline using date-diff analysis.
